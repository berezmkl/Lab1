## 🎯 Мета роботи
Навчитися використовувати стандарти ISO/IEC 25010 та ISO/IEC 5019:2023 для створення моделей якості. Ознайомитися з базовими метриками якості коду і їх застосування у процесі розробки програмного забезпечення.

## 💻 Опис проєкту: «Особистий блог»
* **Тема:** Сайт для ведення персонального блогу з підтримкою постів, коментарів та категорій.
* **Функціонал:** Розділ з інформацією про автора, інтеграція з соціальними мережами.
* **Мета:** Оцінка якості продукту відповідно до стандарту ISO/IEC 25010 та визначення рівня відповідності системи основним характеристикам якості.

## ❓ Відповіді на контрольні питання
1. **Що таке функціональна придатність і як вона оцінюється?** Це здатність ПЗ виконувати визначені функції, що задовольняють потреби користувачів. Оцінюється через відсоток успішно реалізованих функцій за формулою: `(реалізовані / заявлені) × 100%` (ціль ≥ 95%). Вона є фундаментом якості.
2. **Які характеристики ISO/IEC 25010 найважче оцінити?** Зручність використання (*Usability*) та задоволеність через їхній суб'єктивний характер, що вимагає залучення реальних користувачів (наприклад, SUS-шкала) та складно піддається автоматизації. Також важко оцінити повну безпеку, бо нові вразливості з'являються постійно.
3. **Як характеристики якості впливають на сприйняття продукту?** Зовнішні характеристики (швидкість, юзабіліті) формують перше враження та лояльність читачів. Повільне завантаження збільшує відсоток відмов (*bounce rate*), а дефекти безпеки або збої надійності безповоротно нищать репутацію автора.
4. **Як вимірювання характеристик використовуються в процесі розробки?** Під час розробки вони допомагають виявляти проблеми на ранніх етапах через CI/CD (метрики коду, e2e тести). Під час підтримки зібрані метрики (Uptime, Core Web Vitals) сигналізують про потребу негайного реагування або проведення рефакторингу.
